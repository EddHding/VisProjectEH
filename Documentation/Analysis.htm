Background:

My Brother and his girlfriend run a retail shop where they buy and resell vintage and handmade Indian clothes and accessories online and at festivals. Currently all of his taxes, profits and losses etc. are done via an excel spreadsheet; they have recognized that as their business grows they should move to a better system. Therefore I plan to design a double entry book keeping system tailored towards their needs and requirements. Currently there are commercial options which do the same things as I plan to do which can be bought online, however this is something which I wanted to try myself. Some examples of these commercial options are Sage, Kashflow and QuickBooks.

The Goal:

I intend to create a double entry bookkeeping system, it will need to contain everything required for a double entry bookkeeping system to functions, for example it will need an ‘account’ (class) for each asset, liability or capital for the company, each of these ‘accounts’ will need both a credit and debit side (field), this will allow for the user to fill in the required fields when transactions are made, this is the basis of double entry bookkeeping. It will also require a balance sheet which will be linked to these accounts. The system will also need to be able to classify and summarize data into useful easy to locate outputs to give basic information to the user such as the profit/loss.

Users and Stakeholders:

The user and stakeholder of the program will both be my brother and/or his partner, therefore I will only be needing one class of user as it will be a rather basic system, however if I were to develop the program further for a larger business I would require extra user classes with different permissions to cater for the different accountants etc. which would be using the system. However for security reasons I will still include a login system.

Business Benefits:

The intended benefits of my system are relatively simple, as my brother is currently working with an excel spreadsheet I am hoping to simply upgrade this to a more ‘proper’ format, I also want to provide a system which is easier to use and will do most of the work for him as he does not have his own accountant. It will also provide more usability than a spreadsheet will as there will be stock lists and it will provide useful data about the company’s profits/losses and more.

Use Cases:

·        “As the user I would like to be able to view a balance sheet of the company”.

·        “As the user I would like to be able to view the company’s profit and loss statement”

·        “I would like to be able to see a list of all our stock including what it is, the quantity of said item and the price we bought it at”

·        “I would like to have an interface with a drop down menu which will allow me to select an item which is being sold and enter its sale price then have the system automatically update to accommodate this change”
